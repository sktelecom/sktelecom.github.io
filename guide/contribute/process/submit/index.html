кё°м—¬ м ңм¶ң м„ёл¶Җ м Ҳм°Ё

кё°м—¬лҘј м ңм¶ңн•ҳлҠ” м„ёл¶Җ м Ҳм°ЁлҘј м„ӨлӘ…н•ңлӢӨ.
Reference

мқҙ нҺҳмқҙм§ҖлҠ” GitHubмқҳ Open Source Guideмқҳ лӮҙмҡ©мқ„ м°ёкі н•ҳмҳҖлӢӨ.

мқҙм ң кё°м—¬лҘј м ңм¶ңн•ҳлҠ” л°©лІ•мқ„ м•Ңм•„ліҙмһҗ. мқјл°ҳм Ғмқё мҳӨн”ҲмҶҢмҠӨ н”„лЎңм қнҠём—җ кё°м—¬ м ңм¶ң л°©лІ•кіј м Ҳм°ЁлҠ” лӢӨмқҢкіј к°ҷлӢӨ.

1. мқҙм „ мқҙл Ҙмқ„ нҷ•мқён•ҳлқј

м ңм¶ңн•ҳл Өкі  н•ҳлҠ” кё°м—¬к°Җ мқҙм „м—җ лӢӨлӨ„м§„ мқҙл Ҙмқҙ мһҲлҠ”м§Җ нҷ•мқён•ҳлқј. н”„лЎңм қнҠёмқҳ README, мқҙмҠҲ, л©”мқјл§Ғ лҰ¬мҠӨнҠёлҘј мӮҙнҺҙлҙҗлқј. лӘЁл“  л¬ём„ңлҘј лӢӨ нҷ•мқён•  н•„мҡ” м—Ҷмқҙ, лӘҮ к°Җм§Җ нӮӨмӣҢл“ңлҘј кІҖмғүн•ҳл©ҙ мүҪкІҢ нҷ•мқён•  мҲҳ мһҲлӢӨ.

мқҙм „ мһҗлЈҢм—җм„ң кҙҖл Ё лӮҙмҡ©мқ„ м°ҫмқ„ мҲҳ м—ҶлӢӨл©ҙ мқҙмҠҲлҘј м—ҙкұ°лӮҳ Pull RequestлҘј нҶөн•ҙ м»Өл®ӨлӢҲмјҖмқҙм…ҳмқ„ мӢңмһ‘н•  лӢЁкі„мқҙлӢӨ. GitHubм—җм„ңлҠ” IssuesмҷҖ Pull Request кё°лҠҘмқ„ м ңкіөн•ңлӢӨ.

  • Issues : лҢҖнҷ”лӮҳ нҶ лЎ мқ„ мӢңмһ‘н•  мҲҳ мһҲлӢӨ.
  • Pull Request : л¬ём ңм—җ лҢҖн•ң мҶ”лЈЁм…ҳмқ„ кё°м—¬н•ңлӢӨ.

Issue лҳҗлҠ” Pull RequestлҘј м—ҙкё° м „м—җ н”„лЎңм қнҠёк°Җ м ңкіөн•ҳлҠ” л¬ём„ң (мқјл°ҳм ҒмңјлЎң CONTRIBUTING лҳҗлҠ” README)лҘј нҷ•мқён•ҳм—¬ кё°м—¬ м Ҳм°Ё л°Ҹ л°©лІ•мқ„ нҷ•мқён•ҳлқј. мҳҲлҘј л“Өм–ҙ нҠ№м • н…ңн”ҢлҰҝмқ„ л”°лҘҙлҸ„лЎқ мҡ”кө¬н•ҳкұ°лӮҳ, мӮ¬м „ н…ҢмҠӨнҠёлҘј мҡ”кө¬н•  мҲҳ мһҲлӢӨ.

кё°м—¬лҘј мң„н•ң мһ‘м—…мқ„ мӢңмһ‘н•ҳкё° м „м—җ лЁјм Җ IssuesлҘј мҳӨн”Ҳн•ҙм„ң м»Өл®ӨлӢҲнӢ° кө¬м„ұмӣҗм—җкІҢ лЁјм Җ м–ҙл–Ө мһ‘м—…мқ„ н•ҳл Өкі  н•ҳлҠ”м§Җ м•ҢлҰ¬лҠ” кІғлҸ„ мўӢмқҖ л°©лІ•мқҙлӢӨ. л•Ңм—җ л”°лқј л¶Ҳн•„мҡ”н•ң мһ‘м—…мқ„ м§„н–үн•ҳм§Җ м•ҠлҸ„лЎқ лҸ„мӣҖмқ„ л°ӣмқ„ мҲҳ мһҲлӢӨ.

2. IssueлҘј мғқм„ұн•ҳлқј

вҖҢмқјл°ҳм ҒмңјлЎң лӢӨмқҢ мғҒнҷ©м—җм„ң IssueлҘј мғқм„ұн•ңлӢӨ.

  • мҠӨмҠӨлЎң н•ҙкІ°н•  мҲҳ м—ҶлҠ” мҳӨлҘҳ ліҙкі 
  • мғҲлЎңмҡҙ кё°лҠҘ лҳҗлҠ” м•„мқҙл””м–ҙ м ңм•Ҳ
  • м»Өл®ӨлӢҲнӢ° л№„м „, лҳҗлҠ” м •мұ…м—җ лҢҖн•ң нҶ лЎ 
м»Өл®ӨлӢҲмјҖмқҙм…ҳ Tip
  1. лӢӨлЈЁкі мһҗ н•ҳлҠ” мҳӨн”Ҳлҗң Issueк°Җ мһҲлӢӨл©ҙ, лЁјм Җ commentлҘј лӮЁкІЁм„ң лӢӨлҘё мӮ¬лһҢл“Өмқҙ мӨ‘ліөмңјлЎң мһ‘м—…н•ҳм§Җ м•ҠкІҢ н•ҳлқј.
  2. мҳӨлһҳм „м—җ мҳӨн”Ҳлҗң Issueлқјл©ҙ, мқҙлҜё н•ҙкІ°лҗң кІғмқј мҲҳ мһҲлӢӨ. мһ‘м—…мқ„ мӢңмһ‘н•ҳкё° м „м—җ commentлЎң н•ҙкІ°мқҙ лҗң кІғмқҖ м•„лӢҢм§Җ нҷ•мқён•ҳлқј.
  3. IssueлҘј мҳӨн”Ҳн–Ҳм§Җл§Ң, лӮҳмӨ‘м—җ мҠӨмҠӨлЎң лӢөмқ„ м•Ңм•„лӮё кІҪмҡ°, н•ҙлӢ№ Issueм—җ лҢҖн•ң лӢөмқ„ лӢӨлҘё мӮ¬лһҢлҸ„ м•Ң мҲҳ мһҲлҸ„лЎқ commentлҘј мһ‘м„ұн•ҳкі  мқҙмҠҲлҘј close н•ҳлқј. мқҙл ҮкІҢ л¬ём„ңнҷ”н•ҳлҠ” кІғмЎ°м°ЁлҸ„ н”„лЎңм қнҠём—җ кё°м—¬н•ҳлҠ” кІғмқҙлӢӨ.

3. Pull RequestлҘј мғқм„ұн•ҳлқј

кё°м—¬н•  нҢҢмқјмқҙ лӘЁл‘җ мӨҖл№„лҗҳм—ҲлӢӨл©ҙ, Pull RequestлЎң кё°м—¬лҘј м ңм¶ңн•ҳлқј.

Pull Request мӢңкё°

вҖҢмқјл°ҳм ҒмңјлЎң лӢӨмқҢ мғҒнҷ©м—җм„ң Pull RequestлҘј мғқм„ұн•ңлӢӨ. вҖҢ

  • мӮ¬мҶҢн•ң мҲҳм • мӮ¬н•ӯ м ңм¶ң (мҳҲ: мҳӨнғҖ, л§ҒнҒ¬ мҳӨлҘҳ л“ұ)
  • Issueм—җм„ң мқҙлҜё л…јмқҳк°Җ лҗң мӮ¬н•ӯм—җ лҢҖн•ң мһ‘м—… мӢңмһ‘

Pull RequestлҠ” мһ‘м—…мқҙ мҷ„лЈҢлҗң мқҙнӣ„м—җ н•ҙм•ј н•ҳлҠ” кІғмқҖ м•„лӢҲлӢӨ. мқјл°ҳм ҒмңјлЎң Pull RequestлҘј мқјм°Қ мҳӨн”Ҳн•ҳм—¬ лӢӨлҘё мӮ¬лһҢл“Өмқҳ н”јл“ңл°ұмқ„ л°ӣлҠ” кІҢ мўӢлӢӨ. м ңлӘ© мӨ„м—җ “WIP” (Work in Progress)лқјкі  н‘ңмӢңн•ҳм—¬ м•„м§Ғ м§„н–ү мӨ‘мқё мһ‘м—…мқҙлқјкі  н‘ңмӢңн•ҳкі , лӮҳмӨ‘м—җ м–ём ңл“ м§Җ лҚ” л§ҺмқҖ Commitмқ„ м¶”к°Җн•  мҲҳ мһҲлӢӨ.

GitHubм—җм„ңмқҳ Pull Request м Ҳм°Ё

GitHubм—җ мһҲлҠ” н”„лЎңм қнҠёлқјл©ҙ Pull RequestлҘј м ңм¶ң мӢң лӢӨмқҢ мӮ¬н•ӯмқ„ м°ёкі н•  мҲҳ мһҲлӢӨ.

Step 1. Fork

Upstream RepositoryлҘј мһҗмӢ мқҳ GitHub кі„м •мңјлЎң Fork н•ңлӢӨ.

Step 2. Clone

Forkн•ң RepositoryлҘј мһҗмӢ мқҳ Local working directoryлЎң Clone н•ңлӢӨ.

$ mkdir -p $working_dir
$ cd $working_dir
$ git clone https://github.com/$user/[repository]

Upstream RepositoryлҘј Remoteм—җ м¶”к°Җн•ңлӢӨ.

$ cd [repository]
$ git remote add upstream https://github.com/[upstream]/[repository]
 
# Confirm that your remotes make sense:
$ git remote -v

Step 3. Create a branch

лЁјм Җ main branchлҘј fetchмҷҖ rebaseн•ҳм—¬ мөңмӢ  мғҒнғңлЎң мң м§Җн•ңлӢӨ.

$ cd $working_dir/[repository]
$ git fetch upstream
$ git checkout main
$ git rebase upstream/main

к·ёлҰ¬кі  к°ңл°ңмҡ© branch (myfeature)лҘј мғқм„ұн•ңлӢӨ.

$ git checkout -b myfeature

Step 4. Keep your branch in sync

BranchлҘј fetchмҷҖ rebaseн•ҳм—¬ мөңмӢ  мғҒнғңлЎң мң м§Җн•ңлӢӨ.

# While on your myfeature branch
$ git fetch upstream
$ git rebase upstream/main

к·ё мғҒнғңм—җм„ң code мһ‘м—…мқ„ н•ңлӢӨ.

Step 5. Commit

мҲҳм • мӮ¬н•ӯмқ„ commit н•ңлӢӨ.

$ git commit -a -m '[commit message]'

Step 6. Push

myfeature branchмқҳ мҲҳм • мӮ¬н•ӯмқ„ мһҗмӢ мқҳ GitHub Repositoryм—җ Pushн•ңлӢӨ.

git push -f origin myfeature

Step 7. Create a pull request

GitHubм—җм„ң мһҗмӢ мқҳ Repositoryм—җ к°Җл©ҙ Compare & pull request лІ„нҠјмқҙ нҷңм„ұнҷ” лҗң кІғмқ„ ліј мҲҳ мһҲлӢӨ. мқҙлҘј лҲҢлҹ¬м„ң Pull RequestлҘј мғқм„ұн•ңлӢӨ.

мқҙнӣ„ Upstream Repositoryмқҳ кҙҖлҰ¬мһҗлҠ” мҡ”мІӯлҗң Pull RequestлҘј кІҖнҶ н•ҳм—¬ Merge м—¬л¶ҖлҘј кІ°м •н•ңлӢӨ.

м°ёкі  мһҗлЈҢ
  • Pull Requestк°Җ мІҳмқҢмқҙлқјл©ҙ Make a Pull Request(л№„л””мҳӨ к°•мқҳ)лҘј м°ёкі н•ҳлқј. лҳҗн•ң, First Contributions м—җм„ң Pull Request л§Ңл“ңлҠ” кІғмқ„ м—°мҠөн•  мҲҳ мһҲлӢӨ.
  • м°ёкі лЎң, KubernetesлҠ” лӢӨмқҢкіј к°ҷмқҖ Github workflowм—җ лҢҖн•ң м„ӨлӘ… л¬ём„ңлҘј м ңкіөн•ңлӢӨ. : github_workflow.md

4. Feedbackмқ„ л°ӣм•„лқј

кё°м—¬лҘј м ңм¶ңн•ҳл©ҙ н”„лЎңм қнҠёлЎңл¶Җн„° Feedbackмқ„ л°ӣлҠ”лӢӨ. мқјл°ҳм ҒмңјлЎң FeedbackмқҖ к°ңм„ мқҙлӮҳ ліҖкІҪ мӮ¬н•ӯмқҙ м–ҙл–Ө л°©мӢқмңјлЎң лҸҷмһ‘н•ҳлҠ”м§Җ, мҷң к·ёлҹ° л°©мӢқмқ„ мұ„нғқн–ҲлҠ”м§Җ л“ұм—җ лҢҖн•ң м„ӨлӘ…мқ„ мҡ”кө¬н•ңлӢӨ. мқҙ FeedbackмқҖ л•ҢлЎңлҠ” лҢҖмқ‘н•ҳкё° м–ҙл Өмҡё мҲҳлҸ„ мһҲм§Җл§Ң, кё°м—¬мқҳ мҲҳмӨҖмқ„ лҶ’мқҙлҠ” кіјм •мқҙлқјкі  л°ӣм•„л“ӨмқҙлҠ” кІғмқҙ мўӢлӢӨ.

FeedbackмқҖ ліҙнҶө лӢӨмқҢ л„Ө к°Җм§Җ мӨ‘ н•ҳлӮҳм—җ н•ҙлӢ№н•ңлӢӨ.

(1) мқ‘лӢөмқҙ м—ҶлӢӨ?

кё°м—¬н•ҳкё° м „м—җ н”„лЎңм қнҠёк°Җ нҷңл°ңн•ңм§Җ лЁјм Җ нҷ•мқён•ҳлҠ” кІҢ мўӢлӢӨ. м–ҙлҠҗ м •лҸ„ нҷңл°ңн•ң н”„лЎңм қнҠём—җм„ңлҸ„ кё°м—¬м—җ лҢҖн•ҙ мқ‘лӢөмқ„ л°ӣм§Җ лӘ»н•  мҲҳлҸ„ мһҲлӢӨ.

мқјмЈјмқј мқҙмғҒ мқ‘лӢөмқ„ л°ӣм§Җ лӘ»н•ң кІҪмҡ°, лҸҷмқјн•ң мҠӨл Ҳл“ңм—җ м •мӨ‘н•ҳкІҢ лҲ„кө°к°Җм—җкІҢ кІҖнҶ лҘј мҡ”мІӯн•ҳлҠ” кІғмқҙ мўӢлӢӨ. м Ғм Ҳн•ң мӮ¬лһҢмқҳ мқҙлҰ„мқ„ м•Ңкі  мһҲлӢӨл©ҙ @-л©ҳм…ҳ кё°лҠҘмқ„ мқҙмҡ©н•ҳлқј.

Warning

лӢЁ, к°ңмқём ҒмңјлЎң м—°лқҪн•ҳм§ҖлҠ” л§Ҳлқј. мҳӨн”ҲмҶҢмҠӨ н”„лЎңм қнҠём—җм„ң кіөк°ң м»Өл®ӨлӢҲмјҖмқҙм…ҳмқҖ н•„мҲҳмқҙлӢӨ.
